# 🐾 Pet Shop Database - MySQL

## About the Project: This project was developed as part of the **Fundamentals of Database Systems** course.

The objective was to design and implement a complete relational database for a fictional Pet Shop using **MySQL**, applying database design principles and SQL concepts learned throughout the course. The project includes database modeling, table creation, data insertion, SQL queries, and report generation through views. 

Technologies: MySQL, SQL

By Eduarda Liz and Gabriela Bento

---

## 📂 Database Structure

The database was designed to manage the operations of a Pet Shop, including entities and relationships such as customers, pets, employees, services, products, sales, and appointments.

The implementation includes:

* Primary Keys
* Foreign Keys
* Appropriate data types
* Relational database modeling

---

## Features

This project includes:

* Database creation (`CREATE DATABASE`)
* Table creation (`CREATE TABLE`)
* Data insertion (`INSERT INTO`)
* Data filtering and sorting
* SQL queries using:

  * `SELECT`
  * `WHERE`
  * `ORDER BY`
  * `INNER JOIN`
  * `LEFT JOIN`
  * Aggregate functions (`COUNT`, `SUM`, `AVG`, `MAX`, `MIN`)
  * `GROUP BY`
  * `HAVING`
* Creation of a reporting view (`VIEW`)

---

1. Open **MySQL Workbench** (or another MySQL-compatible client).
2. Run the `Petshop.sql` script.
3. The script will:

   * Create the database
   * Create all tables
   * Insert sample data
   * Execute the required SQL queries
   * Create the final reporting view
